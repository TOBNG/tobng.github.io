hexo相关

发表于 2018-03-26 | 分类于 Memo

虽然大一装过一次，但是当时是完全照着网上教程一步一步来的。这次第二次安装。当然也是完全按照网上来的， 但也稍微有点体会吧。

hexo是啥

Hexo is a fast, simple&powerful blog framework

hexo homepage

大概就是一个用以生成静态网页的博客框架，把你写的markdown文章变成html静态网页，其中主题啦渲染啦都是这个框架提供的可选项。

安装hexo的步骤

  1. 安装git
  2. 安装node.js
  3. 安装hexo

git

Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

git homepage

免费开源多平台分布版本管理系统

  • 安装
    官网下载安装
  • git能干嘛
    git（好像）是通过保存文件快照的方式保存每个版本，以此实现版本控制。
  • 分支
    这个东西好像是git的一个特色，然而并没有看懂。实际上，就连仓库是什么也还没完全搞清楚。
  • 和GitHub有什么关系
    git可以本地使用，只要在目录下git init就能在本地建立仓库，而GitHub是一个使用git进行版本控制的…的什么来着？反正就是一个远程的东西，可以在GitHub上建立仓库然后和本地的仓库连接，然后就能pull push之类的。
  • 为什么要先装git
    首先博客要部署到gitpage上是需要github的，其部署方式是git。其次之后还需要用git来pull那些开源的东西。

    这么高端的东西反正目前还没有看懂，先有个大概，懂点了再更。。

node.js

一个js运行环境，生成博客是依赖node.js的。
node.js homepage
当然node.js能做的不止生成博客 可我学不会啊

  • 安装
    官网下载即可，注意到有很多很多版本，总归是装最新的，但是新老版本之间似乎并不完全兼容。
    hexo官网上推荐使用nvm (node version manager?)安装，记得第一次装的时候就是用的这个然后搞了半天…
  • npm
    npm是node.js的包管理器，是全球最大的开源库生态系统。（大概就是可以用来安装开源的包）

安装hexo

装完git和node.js可以装hexo本体了
npm install -g hexo-cli
装完选个目录初始化
hexo init
初始化完之后就瞎几把配置一下，配置文件即hexo目录下的config.yml
再装个next主题
git clone https://github.com/theme-next/hexo-theme-next themes/next
主题的配置文件也是config.yml，在hexo/themes/next目录下
为什么要装这个主题呢，因为用这个主题的人非常多，功能也很完善，有问题一般能搜到解决方案x

常用hexo命令

help

最重要的命令233
$ hexo help

new

创建页面及文章
$ hexo new [layout] <title>

  • layout: post, page等等
  • title即文件名

generate

生成静态文件
$ hexo generate
可以缩写为 你G
hexo g

server

启动本地服务器，默认情况下，访问网址为：http://localhost:4000/
$ hexo server

deploy

部署，根据配置文件中的deploy
$ hexo deploy
缩写为
hexo d

clean

清除已生成的文件，出了什么问题先来一发clean
$ hexo clean

博客部署到GitHub

通过以上的步骤博客已经能在本地运行了但是不能给别人看有什么用于是就要使用GitHub了，GitHub有一个gitpage可以放静态网页，网址 https://username.github.io

  1. 首先要有一个GitHub账户（废话）
  2. 然后用这个账户创一个仓库(repo)叫 username.github.io
  3. 进这个仓库的选项(setting)里开启gitpage设置

ssh

为了从本地部署到github，需要验证账户，或者说登陆。在git push的行为中，为了省去每次登陆的步骤，可以使用通信协议，比如ssh，https，一般来说，ssh更快。
绑定协议后，就可以不用登陆直接推送到远程仓库，绑定的对象是git生成的ssh key和个人GitHub账号。

  • 生成ssh key
    ssh-keygen -t rsa
    生成密钥后，在路径C:\Documents and Settings\username\\.ssh下会生成id_rsa和id_rsa.pub分别对应密钥和公钥。
    将公钥id_rsa.pub的内容添加到GitHub的ssh中即可。
    通过命令 ssh -T git@github.com 进行测试。

deploy

deploy之前是肯定要在配置文件中改一下配置的，但是改完配置之后仍需安装插件。
npm install hexo-deployer-git --save
（大概）也能不安装？

写文章

写文章当然是用markdown编辑器来写，但是也有一些小问题。

设置markdown默认编辑器

windows默认是没有.md这一文件后缀的，因此无法关联默认应用。那只要在注册表里添加.md即可: regedit 打开注册表编辑器，HKEYS_CLASSES_ROOT下新建项.md，然后就能在文件关联选项中看到.md类型了。
msdn solution.aspx#fa_register_type)

插入本地图片

链接引用图片的方式已经了解了，那现在的问题是，如果有一张本地图片，然后我要把插入博客文章中，那为了实现之，我可以把它丢到某个仓库里，然后引用仓库的链接，这样不是很直观，归档不够明确，可以装个插件实现直接丢到_post目录下进行引用（又感觉好像不是必须装插件）
插
npm install hexo-asset-image --save
装完之后还要把博客的_config.yml中post_asset_folder字段改为true
然后在_post文件夹下创建与文章同名的文件夹，就可以直接引用了：
